PAGI

Perl Asynchronous Server Gateway Interface

Bringing Modern Async Web Development to Perl

A spiritual successor to PSGI, inspired by Python's ASGI

The Problem with PSGI

PSGI Limitations

  • Synchronous Only: Single request/response cycle
  • No Long-lived Connections: Can't handle WebSockets, SSE
  • Limited Async Support: psgi.nonblocking is a hack
  • Protocol Bound: Tied to HTTP request/response model
# Traditional PSGI Application
my $app = sub {
    my $env = shift;
    
    # Synchronous processing only
    my $response = process_request($env);
    
    # Must return immediately
    return [200, ['Content-Type' => 'text/plain'], [$response]];
};

# No way to handle:
# - WebSocket connections
# - Server-sent events
# - Long-polling
# - Background tasks

PSGI WebSocket Workarounds

Hacky Solutions

Attempts to add WebSocket support to PSGI required breaking the core specification:

# Plack::App::WebSocket - Not really PSGI compliant
use Plack::App::WebSocket;

my $app = Plack::App::WebSocket->new(
    on_establish => sub {
        my $conn = shift;
        # Connection established, but...
        # - Breaks PSGI's single-call model
        # - Requires server-specific extensions
        # - Not portable across PSGI servers
    },
    on_message => sub {
        my ($conn, $msg) = @_;
        # Handle message, but this is outside PSGI spec
    },
);

# This isn't really PSGI anymore!

What is ASGI?

Python's Solution: ASGI

  • Asynchronous: Built for async/await from the ground up
  • Protocol Agnostic: HTTP, WebSocket, SSE, and more
  • Event-Driven: Message-based communication
  • Long-lived: Applications persist for connection lifetime
# ASGI Application Structure
async def application(scope, receive, send):
    # scope: connection metadata
    # receive: async callable for incoming events
    # send: async callable for outgoing events
    
    if scope['type'] == 'http':
        await handle_http(scope, receive, send)
    elif scope['type'] == 'websocket':
        await handle_websocket(scope, receive, send)
    else:
        raise NotImplementedError(f"Unknown scope type: {scope['type']}")

ASGI HTTP Example

# Python ASGI HTTP Application
async def app(scope, receive, send):
    if scope['type'] == 'http':
        # Wait for request body
        message = await receive()
        
        # Send response headers
        await send({
            'type': 'http.response.start',
            'status': 200,
            'headers': [[b'content-type', b'text/plain']],
        })
        
        # Send response body
        await send({
            'type': 'http.response.body',
            'body': b'Hello from ASGI!',
        })
    else:
        raise NotImplementedError()

ASGI WebSocket Example

# Python ASGI WebSocket Application
async def websocket_app(scope, receive, send):
    if scope['type'] == 'websocket':
        # Wait for connection
        message = await receive()
        assert message['type'] == 'websocket.connect'
        
        # Accept connection
        await send({'type': 'websocket.accept'})
        
        # Echo messages
        while True:
            message = await receive()
            if message['type'] == 'websocket.receive':
                await send({
                    'type': 'websocket.send',
                    'text': f"Echo: {message['text']}"
                })
            elif message['type'] == 'websocket.disconnect':
                break

Introducing PAGI

ASGI for Perl

  • Modern Perl: Uses Future::AsyncAwait (Perl 5.16+)
  • ASGI Compatible: Direct port of ASGI concepts
  • PSGI Backwards Compatible: Adaptation layer included
  • Protocol Support: HTTP, WebSocket, SSE, Lifespan

PSGI

  • Synchronous
  • HTTP only
  • Single call
  • Limited async

PAGI

  • Asynchronous
  • Multi-protocol
  • Long-lived
  • Native async

PAGI HTTP Example

use v5.36;
use Future::AsyncAwait;

# PAGI HTTP Application
async sub application ($scope, $receive, $send) {
    if ($scope->{type} eq 'http') {
        # Wait for request body
        my $message = await $receive->();
        
        # Send response headers
        await $send->({
            type    => 'http.response.start',
            status  => 200,
            headers => [['content-type', 'text/plain']],
        });
        
        # Send response body
        await $send->({
            type => 'http.response.body',
            body => 'Hello from PAGI!',
            more => 0,
        });
    } else {
        die "Unsupported protocol: $scope->{type}";
    }
}

PAGI WebSocket Example

use v5.36;
use Future::AsyncAwait;

# PAGI WebSocket Application
async sub websocket_app ($scope, $receive, $send) {
    if ($scope->{type} eq 'websocket') {
        # Wait for connection
        my $message = await $receive->();
        die unless $message->{type} eq 'websocket.connect';
        
        # Accept connection
        await $send->({type => 'websocket.accept'});
        
        # Echo messages
        while (1) {
            my $message = await $receive->();
            
            if ($message->{type} eq 'websocket.receive') {
                await $send->({
                    type => 'websocket.send',
                    text => "Echo: $message->{text}",
                });
            } elsif ($message->{type} eq 'websocket.disconnect') {
                last;
            }
        }
    } else {
        die "Unsupported protocol: $scope->{type}";
    }
}

PAGI Features

Core Protocols

  • HTTP/1.1 & HTTP/2: Full async support
  • WebSocket: Bidirectional communication
  • Server-Sent Events: Real-time streaming
  • Lifespan: Startup/shutdown hooks

Extensions

  • TLS: Certificate information
  • Custom Protocols: Extensible framework

Implementation

  • IO::Async: Event loop integration
  • Future::AsyncAwait: Modern async syntax
  • Streaming: Efficient memory usage
  • Middleware: Composable applications

Compatibility

  • PSGI Bridge: Run legacy apps
  • Multiple Servers: Pluggable architecture
